Final Products

A dated, indexed shelf of finished, verified work — grouped by project and linked back to its source.

How it works

  • The archivist agent verifies a completed artifact, names it YYYY-MM-DD_type.ext, and files it in its project subfolder.
  • Each project folder has an INDEX.md artifact manifest. The root INDEX.md is the project registry.
  • Nothing lands here until it's done and checked — this folder stays a trustworthy record of output.
Authority is scoped: the archivist works only inside Final_Products/ and writes only on explicit instruction.
