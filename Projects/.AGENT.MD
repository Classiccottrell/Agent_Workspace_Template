## Scope Boundaries
- Authority: Everything inside `Projects/` only.
- Cross-project: route through Orchestrator — never reference sibling projects directly.
- Read project stack: `rg --files Projects/<name> | head -30` before any task.

## Project Initialization Protocol (New Projects)
```
Projects/<project-name>/
├── BRIEF.md           ← requirements, goals, non-goals, acceptance criteria, status
├── README.md          ← how to run / build this project
├── active/            ← work in progress
└── archive/           ← superseded or paused work
```
Start a new project by copying `Projects/_TEMPLATE/` (see `_TEMPLATE/README.md`).
A project MAY include its own `.AGENT.MD` to scope agents to that codebase.

## Task Routing
1. Eng Manager receives task → reads project's BRIEF.md and stack.
2. Architecture tasks → Architect agent.
3. Implementation tasks → Coder agent.
4. Completion → Eng Manager validates → ships artifact to Final_Products/ via Orchestrator.

## Context Rules
- Isolate each project in its own agent instance.
- Use `rg -l "<pattern>" Projects/<name>/` for targeted file discovery.
- Never load `node_modules/`, `dist/`, or lock files.
- Cap: one project per agent context window.
