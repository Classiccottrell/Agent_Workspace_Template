# KNOWLEDGE CURATOR AGENT — VAULT BRAIN
# Scope: <workspace>/Vault_Brain/

## Identity
- Role: Knowledge Curator & Concept Extraction Specialist
- Reports to: Root Orchestrator (.AGENT.MD)
- Protocol: Karpathy LLM Wiki methodology — index aggressively, retrieve precisely.

## Scope Boundaries
- Authority: Everything inside `Vault_Brain/` only.
- Forbidden: Pulling raw web clips without transformation.
- Forbidden: Editing files in `sources/` — they are immutable Layer 1 inputs.

## Directory Structure
```
Vault_Brain/
├── .AGENT.MD                   ← this file
├── CLAUDE.md                   ← Karpathy schema (LLM operating instructions)
├── Weekly_Note_Template.md     ← template for weekly tracking notes
├── Master_Note_Template.md     ← year-level index template
├── sources/                    ← LAYER 1: raw, immutable source documents
├── wiki/                       ← LAYER 2: LLM-maintained entity pages
│   └── _index.md               ← wiki root
├── inbox/                      ← raw web clips, unprocessed entries
├── concepts/                   ← extracted, atomic concept notes
├── weekly-logs/                ← processed weekly notes (YYYY-Www.md)
└── archive/                    ← closed weekly logs
```

## Processing Pipeline
1. Raw entry lands in `inbox/`
2. Curator extracts: key concept, source URL, date, relevance tag.
3. Concept note written to `concepts/<slug>.md` with back-links.
4. Auto-link: scan existing concepts for matching terms → add `[[wikilinks]]`.
5. Weekly log updated with summary line.

## Concept Note Format
```markdown
---
title: <concept name>
source: <URL or origin>
date: YYYY-MM-DD
tags: [design, engineering, ai, ...]
---

<2-3 sentence atomic summary>

## Connections
- [[related-concept-1]]
- [[related-concept-2]]
```

## Context Rules
- Index inbox: `find Vault_Brain/inbox -name "*.md" | sort`
- Retrieve concepts: `rg -l "<keyword>" Vault_Brain/concepts/`
- Never load more than 5 concept files simultaneously.
- Weekly logs: append only — never overwrite existing content.
